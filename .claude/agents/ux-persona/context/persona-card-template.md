# Persona Card Template

Use this template for each persona produced in Phase 4 (Distill).

```markdown
# Persona: [Name]
**Type**: Primary / Secondary / Tertiary / Anti-Persona
**Created**: YYYY-MM-DD | **Session**: #N

## Quick Profile
| Field | Value |
|-------|-------|
| **Role / Occupation** | [what they do] |
| **Tech Comfort** | Low / Medium / High |
| **Context of Use** | [where, when, on what device] |
| **Key Behavior** | [the one thing that defines how they interact] |

## Photo Prompt
> [2-sentence visual description for illustration]

## Goals
1. **Primary**: [most important goal]
2. [Secondary goal]
3. [Tertiary goal]

## Pain Points
1. **Core frustration**: [what blocks them today]
2. [Pain 2]
3. [Pain 3]

## Behavioral Patterns
- [Current workaround / how they solve this today]
- [Frequency of interaction]
- [Decision style: impulsive / methodical / delegator]

## Jobs-To-Be-Done
> **When** [situation], **I want to** [motivation], **so I can** [outcome].

1. When ___, I want to ___, so I can ___.
2. When ___, I want to ___, so I can ___.

## Emotional Journey
| Phase | Emotion | Trigger |
|-------|---------|---------|
| Before | [e.g., Frustrated] | [cause] |
| During | [e.g., Focused] | [cause] |
| After | [e.g., Relieved] | [cause] |

## Scenario
> [3-4 sentences: this persona using the product in a concrete situation — context, goal, action, outcome]

## Design Implications
- [What this persona needs that others might not]
- [Accessibility / inclusion consideration]
- [Performance / device constraint]

## Signals for Other Agents
| Agent | Signal |
|-------|--------|
| **PO** | [User stories to write for this persona] |
| **Architect** | [Technical implications: performance, API, accessibility] |
```
