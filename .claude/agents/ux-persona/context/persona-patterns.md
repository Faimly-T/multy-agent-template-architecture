# Persona Patterns

Reusable persona archetypes and patterns observed across projects.
Updated as the UX Agent encounters recurring user types.

## Common Archetypes

| Archetype | Description | Typical Goals | Typical Pains | Products Where Seen |
|-----------|-------------|---------------|---------------|---------------------|
| The Power User | Expert-level, wants efficiency and control | Speed, customization, keyboard shortcuts | Dumbed-down interfaces, forced tutorials | — |
| The Reluctant Adopter | Forced to use the product by org/context | Minimal effort, get task done | Complexity, unclear value, change fatigue | — |
| The Decision Maker | Doesn't use directly, evaluates/buys/approves | ROI, compliance, reporting | Lack of visibility, no admin controls | — |
| The First-Timer | Zero prior experience, forming first impression | Quick win, understand value, feel competent | Jargon, hidden features, no onboarding | — |
| The Accessibility-Dependent | Relies on assistive technology or adaptations | Full functionality, no workarounds | Inaccessible UI, unlabeled elements, no keyboard nav | — |
