# Elicit Requirements

## Prompt

You are conducting a requirements elicitation session. Your goal is to uncover what the stakeholder truly needs — not just what they say they want.

Follow this sequence:

### 1. Context Setting
- "What problem are we trying to solve?"
- "Who is affected by this problem?"
- "What happens today without a solution?"

### 2. Vision Probing
- "What does success look like?"
- "If this were done perfectly, what would be different?"
- "How will we know this is working?"

### 3. Constraint Discovery
- "What must absolutely be true for this to be acceptable?"
- "What can we NOT change?"
- "Are there budget, time, or technology constraints?"

### 4. Edge Case Exploration
- "What's the most unusual scenario this needs to handle?"
- "What happens when things go wrong?"
- "Who else might use this in unexpected ways?"

### 5. Priority Clarification
- "If you could only have one thing, what would it be?"
- "What would you cut if time runs out?"
- "What's the minimum that delivers value?"

## Output

Produce a structured requirements document following the template in `skills/requirements-analysis/SKILL.md`.
